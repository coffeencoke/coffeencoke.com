Mince

Mince is an object data persistence library written in ruby for ruby, it provides an interface to allow you to write ruby apps using hashy_db, which is an in memory hash database written in ruby, MongoDb, or Amazon's DynamoDB.

Mince has a simple API that can be implemented for any database, providing an easy way to decouple your code from your selected database. Empowering you to change databases for your needs. It also is built specifically to separate responsibilities from database behavior and application specific behavior.

This can be used for any ruby application, even command line libraries.

Mince provides APIs for basic needs, such as a way to configure databases, an rspec shared example to define a contract of implementation of interfaces. As well as more complex needs, such as mixins to provide standard behavior for data models, and the same but for application specific models.

There are a number of libraries that provide additional rich features for Mince.

  • MinceMigrator - A library that uses mince to run data migrations
  • HashyDb - A mince interface for an in-memory hash database written in ruby
  • MinceMongoDb - A mince interface for MongoDb
  • MinceDynamoDb - A mince interface for DynamoDb
